## 🎯 核心设计哲学

本架构严格遵循 **CQRS (命令查询职责分离)** 模式，配合 **Go (业务编排)** 与 **Rust (高性能计算)** 的双引擎驱动。

### 代码库策略: Monorepo

所有服务统一管理，确保协议（Proto）与依赖的一致性。

```txt
bifrost-v3/
├── api/                  # gRPC Protobuf 定义 (通用语言)
│   ├── common/           # 共享消息类型

### 服务详解

#### 🌐 Go 服务层（业务编排）

##### Gjallar (API 网关)

- **端口**: 8080 (HTTP)
- **技术**: Go + gRPC Gateway
- **职责**:
  - HTTP API 统一入口
  - 路由分发 (写→Nexus, 读→Beacon, 搜索→Mirror)
  - JWT 鉴权中间件
  - OpenTelemetry 追踪

##### Nexus (写服务)

- **端口**: 9001 (gRPC)
- **技术**: Go + PostgreSQL + NATS
- **职责**:
  - 文章创建/更新
  - **同步调用 Forge 渲染**（5s 超时）
  - 事务内完成 HTML 存储
  - 发布 NATS 事件
  - MinIO 对象存储管理

**关键特性**: 同步渲染确保数据一致性，失败自动回滚事务

##### Beacon (读服务)

- **端口**: 9002 (gRPC)
- **技术**: Go + PostgreSQL + Redis
- **职责**:
  - 文章查询/列表（已渲染的 HTML）
  - Redis 多级缓存
  - 分类/标签查询
  - 评论查询

#### ⚡ Rust 服务层（高性能计算）

##### Forge (渲染引擎)

- **端口**: 9092 (gRPC)
- **技术**: Rust + pulldown-cmark + ammonia
- **职责**:
  - Markdown → HTML 渲染（含代码高亮）
  - TOC (目录) 生成
  - XSS 防护（HTML 清洗）
- **性能**: <100ms/文章，P95 延迟 ~80ms

##### Mirror (搜索引擎)

- **端口**: 9093 (gRPC)
- **技术**: Rust + Tantivy (倒排索引)
- **职责**:
  - 全文索引（title, content, tags, BM25 算法）
  - 实时搜索（P95 < 50ms）
  - 分面统计 & 搜索建议
- **存储**: 持久化索引文件（共享存储或主从同步）

##### Oracle (异步处理)

- **端口**: 9094 (gRPC)
- **技术**: Rust + NATS + Tokio
- **职责**:
  - 消费 NATS 事件（post.created, post.updated）
  - 更新 Mirror 索引
  - 浏览量/点赞数聚合
  - 统计计算
  - 定时任务

## 🔄 数据流

### 写链路（同步渲染 + 异步索引）

```txt
1. 用户提交文章
   ↓
2. Gjallar → Nexus (gRPC)
   ↓
3. Nexus 开启数据库事务
   ↓
4. Nexus → Forge (同步 gRPC，5s 超时) ⚠️ 关键设计点
   ├─ 请求: { raw_markdown: "# Title\n\nContent..." }
   └─ 响应: { html_body: "<h1>Title</h1>...", toc_json: "[...]" }
   ↓
5. Forge 渲染 Markdown → HTML（XSS 清洗 + 代码高亮）
   ↓
6. Nexus 存储到 PostgreSQL (raw_markdown + html_body)
   ↓
7. 提交事务 ✅ (原子性保证)
   ↓
8. 发布 NATS 事件: "content.post.created"
   ↓
9. Oracle 消费事件 → 更新 Mirror 索引 (异步)

```

**🎯 关键设计决策**:

| 设计点 | 策略 | 原因 |
|--------|------|------|
| 渲染时机 | **同步渲染**（事务内） | 确保 `raw_markdown` 和 `html_body` 原子存储 |
| 故障处理 | **5s 超时 + 自动回滚** | Forge 故障不导致脏数据 |
| 索引更新 | **异步**（NATS 事件） | 不阻塞写入主流程，接受最终一致性 |

### 读链路（多级缓存）

```txt
1. 用户请求文章详情
   ↓
2. Gjallar → Beacon (gRPC)
   ↓
3. Beacon 检查 Redis 缓存
   ├─ Hit → 返回缓存数据 (L2 缓存)
   └─ Miss → 继续
       ↓
4. 查询 PostgreSQL
   ├─ 直接读取 html_body (无需渲染)
   └─ 返回完整数据
   ↓
5. 写入 Redis 缓存 (2h TTL)
   ↓
6. 返回给客户端

```

**🗂️ 缓存策略**:

| 层级 | 实现 | TTL | 触发失效 |
|------|------|-----|----------|
| L1 | 本地内存（计划中） | 5min | NATS `post.updated` |
| L2 | Redis | 2 小时 | NATS `post.updated` + LRU 淘汰 |

### 搜索链路（同步调用）

```txt
1. 用户搜索 "golang microservices"
   ↓
2. Gjallar → Mirror (同步 gRPC，5s 超时)
   ├─ 请求: { query: "golang microservices", page: 1, page_size: 20 }
   ↓
3. Mirror 执行 Tantivy 查询
   ├─ 分词: ["golang", "microservices"]
   ├─ 倒排索引查询
   ├─ 相关性排序 (BM25)
   └─ 高亮片段生成
   ↓
4. 返回: {
     hits: [
       { id: 123, title: "...", highlight: "..." },
       ...
     ],
     total: 42,
     took_ms: 15.3
   }
   ↓
5. Gjallar 返回给客户端

```

**🔍 搜索特性**:

| 功能 | 实现 | 性能指标 |
|------|------|----------|
| 全文检索 | Tantivy BM25 算法 | P50: 15ms, P95: 35ms |
| 高亮显示 | 动态片段生成 | - |
| 分面聚合 | 按分类/标签统计 | - |
| 搜索建议 | Fuzzy Match + Prefix | - |

## 🗄️ 数据库设计

> 完整 DDL 详见 [migrations/](../migrations/)

### PostgreSQL 核心表（简化版）

```sql
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL
);

-- 文章表（包含渲染后的 HTML）
CREATE TABLE posts (
    id BIGINT PRIMARY KEY,
    author_id BIGINT REFERENCES users(id),
    title VARCHAR(255) NOT NULL,
    slug VARCHAR(255) UNIQUE NOT NULL,
    raw_markdown TEXT NOT NULL,  -- 原始 Markdown
    html_body TEXT NOT NULL,     -- Forge 渲染的 HTML ⚠️
    toc_json JSONB,              -- 目录 JSON
    status VARCHAR(20) DEFAULT 'draft',
    INDEX idx_slug (slug),
    INDEX idx_status_created (status, created_at)
);

-- 评论表
CREATE TABLE comments (
    id BIGINT PRIMARY KEY,
    post_id BIGINT REFERENCES posts(id),
    user_id BIGINT REFERENCES users(id),
    content TEXT NOT NULL
);
```

## 🔐 安全措施

- **JWT 鉴权**: HS256 签名，刷新令牌机制
- **XSS 防护**: Forge 使用 ammonia 库清洗 HTML
- **SQL 注入防护**: 参数化查询 (pgx)
- **密码加密**: bcrypt (cost=10)
- **HTTPS**: 生产环境强制 TLS
- **CORS**: 配置白名单
- **限流**: 基于 IP 的 Token Bucket
