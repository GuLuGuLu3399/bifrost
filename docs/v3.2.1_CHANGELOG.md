## 🔧 核心代码实现

### Messenger 包

```go
// go_services/internal/pkg/messenger/messenger.go

type Client struct {
    conn *nats.Conn
}

// 核心方法：发送 & 订阅
func (c *Client) Publish(subject string, payload interface{}) error
func (c *Client) Subscribe(subject, group string, handler Handler) (*nats.Subscription, error)
```

**特点**：

- 极简设计（仅 ~50 行代码）
- Fire-and-Forget（异步，不重试）
- 自动错误转换（xerr 集成）

### Beacon Consumer

```go
// go_services/internal/beacon/data/consumer.go

func (c *Consumer) Start() error {
    sub, err := c.msgr.Subscribe("content.>", "beacon_service", c.handleEvent)
    // ...
}

func (c *Consumer) handlePostChange(data []byte) {
    // 删除 Redis 缓存
}
```

**特点**：

- 自动路由事件到处理函数
- Queue Groups 负载均衡
- 宽松错误处理（只记日志）

### Nexus Events

```go
// go_services/internal/nexus/data/events.go

func (d *Data) PublishPostUpdated(ctx context.Context, msgr *messenger.Client, id, slug) {
    go func() {
        msgr.Publish(messenger.SubjectPostUpdated, payload)
    }()
}
```

**特点**：

- 业务成功后异步发送
- 不阻塞主流程
- 发送失败只记日志

## 🎯 设计对比

### Outbox 模式 (旧)

```
业务 → DB事务 → Outbox表 → Relayer轮询 → 重试/锁定 → NATS
```

❌ 复杂、高维护成本、DB压力大

### Fire-and-Forget (新)

```
业务 → DB事务成功 → go func() { NATS发送 }
```

✅ 简洁、零DB压力、易扩展

## 🎓 设计思想

> **以简单御复杂** (Simple is Better)
>
> Bifrost 采用最小化的设计：
>
> - 代码少 = 易维护
> - 依赖少 = 易调试
> - 复杂度低 = 认知负担小
>
> 同时借助 NATS 的力量实现：
>
> - 服务解耦 (Subject 隔离)
> - 自动负载均衡 (Queue Groups)
> - 易于扩展 (新增消费者零成本)
>
> 这正是"系统设计的黄金比例"。
