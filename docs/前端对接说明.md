# 前端对接说明（Backend → Frontend）

> 适用范围：Bifrost v3.2 当前公开与受限接口说明，供 Web 前端（Nuxt/Horizon）接入使用。

---

## 环境与网关

- 基础网关：`http://localhost:8080`
- 协议：HTTP(JSON)，由 Gjallar 通过 gRPC-Gateway 暴露 `/v1/*` 路由
- 编码：UTF-8，`Content-Type: application/json`
- 健康检查（内部）：`/health`

---

## 认证与鉴权

- 认证方式：JWT Bearer Token
- 开放接口：Beacon（读服务）与 Mirror（搜索）多数无需登录
- 需要登录：发表评论、个人资料变更、管理端（文章增删改、分类/标签管理、存储凭证）
- 传递方式：`Authorization: Bearer <access_token>`

登录/注册接口（Nexus/UserService）：

- `POST /v1/auth/login` → 返回 `access_token`, `refresh_token`, `expires_in`
- `POST /v1/auth/register`
- `GET /v1/users/profile`（需登录）
- `PUT /v1/users/profile`（需登录）

## 公开接口（无需登录）

### 内容读取（Beacon/BeaconService）

- 获取文章详情（HTML 渲染版）
  - `GET /v1/posts/{slug}`
  - 响应体：`post`（`PostDetail`）包含：`id,title,slug,summary,cover_image_key,html_body,toc_json,view_count,like_count,published_at,updated_at,author,category,tags,prev_*,next_*`

- 获取文章列表（摘要版）
  - `GET /v1/posts`
  - 查询参数：
    - 分页：`page.page_size`（默认 20，最大 100）、`page.page_token`
    - 过滤：`category_id`、`tag_id`、`author_id`
  - 响应体：`posts[]`（`PostSummary`），`page`（`PageResponse` 包含 `next_page_token,total_count`）

- 批量获取文章摘要（用于搜索结果聚合）
  - `POST /v1/posts:batch`
  - 请求体：`{ "post_ids": [1,2,3] }`
  - 响应体：`posts[]`（`PostSummary`）

- 获取某文章评论列表
  - `GET /v1/posts/{post_id}/comments`
  - 查询参数：`page.page_size`、`page.page_token`、`root_id`（楼中楼）
  - 响应体：`comments[]`（`CommentItem`）、`page`

- 获取用户公开信息
  - `GET /v1/users/{user_id}` → `user`（`UserProfile`）

- 元数据：分类与标签
  - `GET /v1/categories` → `categories[]`
  - `GET /v1/tags` → `tags[]`

字段参考（节选）：

- `AuthorInfo { id,nickname,avatar_key }`
- `CategoryItem { id,name,slug,description,post_count }`
- `TagItem { id,name,slug,post_count }`

### 全文搜索（Mirror/MirrorService）

- 搜索
  - `GET /v1/search`
  - 查询参数：
    - `query`（关键词）
    - 分页：`page.page_size`,`page.page_token`
    - 过滤：`filter.category_id`,`filter.tag_id`,`filter.author_id`,`filter.date_range_start`,`filter.date_range_end`
    - `include_facets`（布尔）
  - 响应体：
    - `hits[] { id,score,title,slug,highlight_title,highlight_content,published_at }`
    - `total_hits`, `took_ms`, `facets`（`categories`,`tags` 的计数字典）

- 搜索建议
  - `GET /v1/search/suggest`（参数：`prefix`,`limit`）
  - 响应体：`suggestions[]`, `took_ms`

## 登录后接口（常用）

- 发表评论（Nexus/CommentService）
  - `POST /v1/posts/{post_id}/comments`
  - 需登录，体：`{ "content": "...", "parent_id": 0 }`
  - 响应：`{ "comment_id": 123, "status": "..." }`

- 删除评论（作者或管理员）
  - `DELETE /v1/comments/{comment_id}`

- 获取上传凭证（直传 MinIO）
  - `POST /v1/storage/upload_ticket`
  - 需登录，请求：`{ "filename": "avatar.png", "usage": "avatar" }`
  - 响应：`{ "upload_url": "...", "object_key": "..." }`

> 前端拿到 `upload_url` 直接 PUT/POST 上传到对象存储；业务数据保存 `object_key`，展示时拼接 CDN。

---

## 管理端（Admin，需登录）

- 文章管理（Nexus/PostService）
  - `GET /v1/drafts`（分页、筛选）
  - `GET /v1/admin/posts/{post_id}`（详情：完整字段）
  - `GET /v1/admin/posts/{post_id}/source`（编辑器用 Markdown 源）
  - `POST /v1/admin/posts`（创建）
  - `PUT /v1/admin/posts/{post_id}`（更新）
  - `DELETE /v1/admin/posts/{post_id}`（删除）

- 分类管理（Create/Update/Delete、`/v1/categories` 路径族）
- 标签清理（`DELETE /v1/tags/{tag_id}`）

---

## 静态资源与 CDN

- 字段 `cover_image_key`、`avatar_key` 为对象存储 Key
- 前端拼接规则（示例）：`https://cdn.example.com/${object_key}`
- 若有反向代理/网关映射，按运维提供的 CDN 基础路径拼接

---

## 分页规范

- 请求：`page.page_size`、`page.page_token`
- 响应：`page.next_page_token`、`page.total_count`
- 建议：
  - 列表页先用 `page_size=20`，滚动加载用 `next_page_token`
  - 搜索建议 `limit` 建议 5~10

## 示例

获取文章列表（首页）：

```bash
curl -s "http://localhost:8080/v1/posts?page.page_size=20"
```

获取文章详情（slug）：

```bash
curl -s "http://localhost:8080/v1/posts/hello-bifrost"
```

搜索：

```bash
curl -s "http://localhost:8080/v1/search?query=rust&page.page_size=10&include_facets=true"
```

发表评论（需登录）：

```bash
curl -s -X POST \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"content":"Great post!","parent_id":0}' \
  "http://localhost:8080/v1/posts/123/comments"
```

直传文件：

```bash
# 1) 获取上传凭证
curl -s -X POST \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"filename":"avatar.png","usage":"avatar"}' \
  "http://localhost:8080/v1/storage/upload_ticket"

# 2) 使用返回的 upload_url 直接上传到对象存储（示例命令因存储而异）
```

## 错误与重试

- HTTP 状态码对齐语义：`400/401/403/404/409/500` 等
- 错误体（示例）：`{ "code": 400, "message": "bad request" }`
- 建议：
  - 登录态过期：捕获 `401`，触发刷新或跳转登录
  - 列表/搜索失败：指数退避重试，提示“稍后重试”
