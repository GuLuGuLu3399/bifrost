## 目录

- [包分层](#包分层)
- [使用建议](#使用建议)

## 包分层

| 包名 | 说明 |
| --- | --- |
| `xerr` | 统一错误码与错误包装 |
| `observability` | 日志、追踪、指标 |
| `network` | HTTP/gRPC 基础设施与中间件 |
| `database` | 数据库连接与基础封装 |
| `cache` | 缓存客户端封装 |
| `security` | JWT 与密码相关能力 |
| `id` | 分布式 ID 能力 |
| `lifecycle` | 生命周期管理 |
| `middleware` | 通用中间件 |

## 使用建议

- 对外错误统一通过 `xerr` 输出。
- 开发环境可提高日志级别，生产环境建议降噪。
- 优先复用统一初始化流程，避免各服务实现漂移。

## 约束

- 仅放跨服务可复用基础能力。
- 不放具体业务逻辑。
